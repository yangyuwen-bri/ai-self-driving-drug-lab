# Demo Guide

这份文档给你自己看，不给演示对象看。

## 页面定位

- `总览`：开场页，只用来介绍这是一个虚拟实验室演示平台
- `前台实验室`：给外部观众看目标 `Y -> X 配方 -> 虚拟实验 -> 结果` 这条主线
- `后台观测室`：给技术或管理方看 run 历史、误差和结果诊断
- `接入路线`：讲未来如何接真实实验室、真实仪器和真实数据

## 你怎么演示

### 1. 先开总览

只讲一句：

> 这是一个自驱动药物实验的虚拟实验室演示平台，先用数字孪生跑通闭环，后续再接真实实验室。

### 2. 切到前台实验室

你重点讲：

- 用户只需要输入目标 `Y`
- 系统会自动生成实验处方 `X`
- 点击后系统自动执行一轮虚拟实验
- 页面返回结果 `Y`
- 如果还没达到目标，系统会自动继续优化

建议演示时把目标半衰期设成 `12h`。

### 3. 跑完后切到后台观测室

你重点讲：

- 每一轮实验都有记录
- 可以看到误差是否在收敛
- 可以看到最佳结果和历史 run
- 未来这里会接真实实验观测值，而不是模拟值

### 4. 最后切到接入路线

你只讲三件事：

- 现在是 simulation-first
- 后续替换实验执行层
- 后续替换观测数据层

## 页面上不要讲的内容

这些内容你自己知道就行，不要在外部演示时展开：

- 当前是 Streamlit 原型
- 代码结构怎么拆
- 哪些模块还是 stub
- 哪些 JSON 字段来自哪里
- BayBE / surrogate 的技术细节

## 演示对象关注点

### 给老板 / 客户

他们只关心：

- 这个事能不能闭环
- 是否能减少试错
- 后面能不能接真实实验室

### 给合作实验室

他们更关心：

- 未来如何接真实实验设备
- 实验数据怎么回流
- 系统如何记录和复用配方

### 给技术团队

他们更关心：

- 前后台怎么分层
- run 历史怎么存
- 模型如何更新
- 接口怎么预留
